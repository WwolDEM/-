# Деловой помощник

Адаптивный одностраничный сайт персонального сервиса цифровой и организационной помощи. Проект работает на обычных HTML, CSS и JavaScript и не требует сборки или backend.

## Локальный запуск

Откройте `index.html` в браузере или запустите локальный сервер из папки проекта:

```bash
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000`.

## Telegram-форма

Подключён контакт `@Trifon_Koss_W`. Форма проверяет обязательные поля, формирует текст обращения и открывает Telegram с корректно закодированным сообщением. Перезагрузки страницы и отправки данных на сервер сайта не происходит.

Если браузер блокирует автоматическое открытие, появляется резервное окно: пользователь может скопировать обращение и открыть Telegram вручную.

## Что обязательно заполнить перед публичным запуском

### Домен и SEO

1. После выбора домена добавить в `<head>` файла `index.html` абсолютный `canonical`.
2. Добавить `og:url` и заменить относительный `og:image` на абсолютный адрес изображения.
3. Добавить адрес главной страницы в `sitemap.xml`.
4. Раскомментировать строку `Sitemap` в `robots.txt` и подставить реальный домен.
5. Не использовать временный или вымышленный домен.

### Политика конфиденциальности

Файл `privacy-policy.template.html` — только неиндексируемый шаблон. Он не считается готовой политикой и не связан с основной страницей.

До публикации владелец должен заполнить и юридически проверить:

- полное имя или наименование оператора;
- актуальный статус исполнителя, если его требуется указывать;
- контакт для вопросов об обработке и удалении данных;
- адрес опубликованного сайта;
- фактически собираемые данные;
- цели и основания обработки;
- реальный порядок и срок хранения;
- способ отзыва согласия и удаления данных;
- сведения о хостинге, аналитике и иных подключённых сервисах;
- роль Telegram как внешнего сервиса, в который пользователь самостоятельно отправляет сообщение;
- дату вступления политики в силу.

Не добавляйте неизвестные паспортные данные, адрес, ИНН или другие реквизиты. После заполнения переименуйте документ в `privacy-policy.html`, уберите `noindex` только после проверки и замените временную подпись в футере на настоящую ссылку.

## Демонстрационные работы

Все четыре примера явно обозначены как демонстрационные и не выдаются за клиентские проекты. Их можно постепенно заменять настоящими работами с разрешения клиентов или обезличенными материалами.

## Проверка после загрузки на хостинг

1. Открыть сайт по HTTPS и проверить отсутствие смешанного содержимого.
2. Проверить все кнопки, якорные ссылки и Telegram-ссылки.
3. Отправить тестовую заявку с телефона и компьютера.
4. Проверить резервное копирование обращения при заблокированном всплывающем окне.
5. Проверить мобильное меню, FAQ и четыре модальных окна.
6. Проверить превью ссылки в Telegram после добавления абсолютных Open Graph URL.
7. Проверить `robots.txt`, `sitemap.xml`, favicon и код ответа 404 для несуществующей страницы.
8. Запустить Lighthouse или PageSpeed Insights на опубликованном адресе.
9. Проверить страницу на реальных устройствах, особенно при ширине около 320–430 px.
10. Подтвердить актуальность условий оплаты, правок, хранения файлов и срочных заказов.
